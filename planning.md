### Tool 1: search_listings

**What it does:**
Searches the mock thrift listings dataset for items that match the user’s requested description, size, and maximum price. It filters listings using the available fields in `listings.json`, then returns the best matching items sorted by relevance.

**Input parameters:**
- `description` (str): The item or style the user is searching for, such as `"vintage graphic tee"` or `"black denim jacket"`.
- `size` (str): The user’s requested size, such as `"S"`, `"M"`, `"L"`, or `None` if the user does not provide a size.
- `max_price` (float): The maximum amount the user wants to spend, such as `30.0`.

**What it returns:**
Returns a list of listing dictionaries. Each dictionary may include fields such as `id`, `title`, `description`, `category`, `style_tags`, `size`, `condition`, `price`, `colors`, `brand`, and `platform`. The list is filtered by description, size, and max price, then sorted so the most relevant results appear first.

**What happens if it fails or returns nothing:**
If no listings match, the tool returns an empty list `[]` instead of crashing. The planning loop checks for an empty list, stores an error message in `session["error"]`, and returns early without calling `suggest_outfit` or `create_fit_card`. The user is told to try a broader description, different size, or higher budget.

---

### Tool 2: suggest_outfit

**What it does:**
Creates an outfit suggestion using the selected thrift listing and the user’s wardrobe. It explains how the new item could be styled with existing wardrobe pieces, or gives general styling advice if the wardrobe is empty.

**Input parameters:**
- `new_item` (dict): The selected listing returned from `search_listings`, usually `results[0]`. It contains item details like title, category, price, size, colors, brand, platform, and style tags.
- `wardrobe` (dict): The user’s wardrobe data following the structure in `wardrobe_schema.json`. It contains wardrobe items that can be used to build an outfit.

**What it returns:**
Returns a string containing one or more outfit suggestions. The suggestion should mention the selected thrift item and explain what wardrobe pieces to pair it with. If possible, it should include styling details such as fit, layering, shoes, colors, or overall aesthetic.

**What happens if it fails or returns nothing:**
If the wardrobe is empty or minimal, the tool should still return a useful general styling suggestion instead of crashing. If no specific outfit can be created, the agent should explain that the wardrobe has limited items and suggest a simple outfit direction based on the new item’s category, color, and style tags.

---

### Tool 3: create_fit_card

**What it does:**
Creates a short, shareable outfit caption based on the selected thrift item and the outfit suggestion. The result should sound like a social media caption, not a plain product description.

**Input parameters:**
- `outfit` (str): The outfit suggestion generated by `suggest_outfit`.
- `new_item` (dict): The selected listing from `search_listings`, including details such as title, price, brand, platform, and style tags.

**What it returns:**
Returns a short social-media-style caption string. The caption should reference the selected item, the styling idea, and purchase details like price or platform when available. It should be different for different inputs and should feel natural enough to share.

**What happens if it fails or returns nothing:**
If the outfit string is empty or the selected item is missing required details, the tool returns a clear error message string instead of crashing. The planning loop stores that message in `session["error"]` or displays it to the user so they know the fit card could not be created.

## Planning Loop

**How does your agent decide which tool to call next?**

The agent starts by reading the user query and extracting the item description, size if provided, and maximum price if provided. It first calls `search_listings(description, size, max_price)`.

After `search_listings` runs, the planning loop checks whether the returned list is empty. If `results == []`, the agent sets `session["error"]` to a message such as `"No matching listings were found. Try a broader description, different size, or higher budget."` Then it returns the session immediately. In this error path, the agent does not call `suggest_outfit` or `create_fit_card`.

If listings are found, the agent selects the first result as the best match and stores it as `session["selected_item"] = results[0]`. Then it calls `suggest_outfit(session["selected_item"], wardrobe)`.

After `suggest_outfit` returns, the agent stores the outfit suggestion as `session["outfit_suggestion"]`. If the outfit suggestion is empty or invalid, the agent stores an error message and returns early.

If the outfit suggestion is valid, the agent calls `create_fit_card(session["outfit_suggestion"], session["selected_item"])`. The result is stored as `session["fit_card"]`. At that point, the workflow is complete, and the agent returns the full session containing the selected item, outfit suggestion, fit card, and any error value.

## State Management

**How does information from one tool get passed to the next?**

The agent uses a `session` dictionary to store information during one user interaction. This allows the output from one tool to become the input for the next tool without asking the user to re-enter the same information.

The main state values are:

- `session["query"]`: The original user query.
- `session["selected_item"]`: The top listing selected from `search_listings`.
- `session["outfit_suggestion"]`: The styling recommendation returned by `suggest_outfit`.
- `session["fit_card"]`: The final caption returned by `create_fit_card`.
- `session["error"]`: Any error message if a tool fails or returns nothing useful.

For example, `search_listings` returns a list of item dictionaries. The planning loop stores the first result in `session["selected_item"]`. Then `suggest_outfit` uses `session["selected_item"]` as its `new_item` input. After that, the result from `suggest_outfit` is stored in `session["outfit_suggestion"]`, which is then passed into `create_fit_card`.

    A[User Query] --> B[Planning Loop]

    B --> C[Extract description, size, max_price]
    C --> D[search_listings description, size, max_price]

    D --> E{Any listings found?}

    E -->|No| F[Set session error]
    F --> G[Return early to user]

    E -->|Yes| H[Set session selected_item = results[0]]

    H --> I[suggest_outfit selected_item, wardrobe]
    I --> J{Valid outfit suggestion?}

    J -->|No| K[Set session error]
    K --> G

    J -->|Yes| L[Set session outfit_suggestion]

    L --> M[create_fit_card outfit_suggestion, selected_item]
    M --> N{Valid fit card?}

    N -->|No| O[Set session error]
    O --> G

    N -->|Yes| P[Set session fit_card]
    P --> Q[Return full session to user]

    R[(Session State)] <--> H
    R <--> L
    R <--> P
    R <--> F
    R <--> K
    R <--> O
